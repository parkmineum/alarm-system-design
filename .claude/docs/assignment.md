# 과제: BE-C 알림 발송 시스템

> 출처: <https://be-assignment-ecru.vercel.app/assignment/MOR7CoynqtQnU8Ak4daf2XwXZ5sl1uvE>
> 유형: 이벤트 / 비동기 / 운영 고려형
> 제출 기한: 첫 접속일 기준 **5일 이내**

## 문제 정의

수강 신청 완료, 결제 확정, 강의 시작 D-1, 취소 처리 등 다양한 이벤트가 발생할 때 사용자에게 이메일 또는 인앱 알림을 발송한다.

- 알림 처리 실패가 비즈니스 트랜잭션에 영향을 주지 않아야 한다
- 중복 발송을 방지해야 한다

---

## 필수 요구사항

### 1. 알림 발송 요청 API

- **등록**: 수신자 ID, 알림 타입, 참조 데이터(이벤트/강의 ID), 채널(`EMAIL` / `IN_APP`)
- **상태 조회**: 특정 알림의 현재 상태
- **목록 조회**: 수신자 기준 알림 목록 (읽음/안읽음 필터 포함)

### 2. 알림 처리 상태 관리

- 발송 실패 시 재시도 정책과 최종 실패 처리 방식 설계
- 실패 사유 기록

### 3. 중복 발송 방지

- 동일 이벤트에 대해 알림이 중복 발송되지 않아야 한다
- 동시에 같은 요청이 여러 번 들어오는 경우도 고려한다

### 4. 비동기 처리 구조

- 알림 발송은 API 요청 스레드와 분리되어 처리되어야 한다
- 실제 메시지 브로커 없이 구현, 단 **실제 운영 환경으로 전환 가능한 구조**여야 한다

### 5. 운영 시나리오 대응

- 처리 중 상태가 일정 시간 이상 지속되는 경우 복구 방법 설계
- 서버 재시작 후에도 미처리 알림이 유실 없이 재처리되어야 한다
- 다중 인스턴스 환경에서도 동일 알림이 중복 처리되지 않아야 한다

---

## 선택 구현 (가산점)

- 발송 스케줄링: 특정 시각에 발송 예약
- 알림 템플릿 관리 (타입별 메시지 템플릿)
- 읽음 처리: 여러 기기에서 동시 읽음 요청 대응
- 최종 실패 알림 보관 및 수동 재시도
